# PickleBallers PWA — Progress Report

Date: June 1, 2026
App: PickleBallers mobile app (PWA)

## Summary

Brought the **venue-owner console** into the mobile app — previously this existed only on the website. Venue owners can now manage their venues end-to-end from the app. It's built against the **same backend the website uses**, so no server changes were needed.

How owners get in: **You tab → “My venues.”** This row only shows for venue owners; regular players don't see it.

## Done today — Venue owner mode

* My venues (owner entry) — venue owners get a “My venues” shortcut in their profile that opens the owner console.
* Your venues — see every venue you own in one place, with quick stats (venues, claimed, verified, total courts), and open any one to manage it.
* Venue overview — check a venue’s key numbers (courts, open days, reviews, rating) and see exactly which listing details are still missing to reach 100%.
* Listing — edit how your venue appears publicly: name, summary, description, indoor/outdoor, surface, contact, pricing, amenities, and highlights.
* Location — pinpoint your venue by searching an address or dragging the map pin, then save the exact spot.
* Hours — set open and close times for each day of the week, mark rest days as closed, and add one-off holiday closures.
* Courts — add new courts and edit, activate/deactivate, or remove existing ones.
* FAQs — write, edit, and delete the questions and answers players see on your listing.
* Reviews — read player reviews and post a reply (one per review); see total reviews and average rating.
* Photos — upload venue photos and see your current hero and gallery.
* Create a new venue — add a brand-new venue with just a name required, then finish it in the editor.
* Preview public listing — jump straight to what players see (hero, tags, courts/surface, map, amenities) without leaving the console.
* Roadmap updated — today’s progress is live at [https://pickleballer.eunika.xyz/roadmap](https://pickleballer.eunika.xyz/roadmap)

* Owner API client added to `app/src/shared/lib/api.ts`; same owner-gated endpoints as the website — no API/server changes.
* `npm run build` and `npm run lint` both clean; every owner endpoint verified against the live API.
* Shipped in commit `2230c48`.

## Still in progress

### Owner console (known gaps, shown honestly in-app)
* Photos — uploading works, but choosing a new main photo, reordering, or deleting isn’t available yet.
* Venue address & city — still managed by the team (owners can move the map pin themselves).
* Long edit sessions — after a while you may be asked to sign in again.
* Bookings — owners can’t see or manage incoming bookings yet.
* Owner analytics — revenue, occupancy, and other stats aren’t there yet.

### Rest of the app (unrelated to the owner work)
* Games & Clubs — still using sample data.
* Search & Notifications — not connected yet.
* Profile saving — edits don’t save yet.
* Map pins & distance search / court filters — still being wired.
