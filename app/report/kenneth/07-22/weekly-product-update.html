The biggest outcomes for players and venues

These are the changes clients and users are most likely to notice.

⏱

Booking requests no longer sit forever

Owner-approval requests now have clear deadlines, countdowns, reminders, automatic release of expired court time, and messages for both player and owner.

✓

Joining Open Play is a complete journey

A player’s pending request now stays visible after leaving the screen, can be cancelled, and can be reviewed by the host without duplicate or over-capacity joins.

⌂

Venue teams have better day-to-day controls

Manual reservations show booked and maintenance hours clearly, fully booked courts are disabled, and owners can give each staff member only the access they need.

↗

The website now leads people into the app

Booking, login, registration, game creation, club creation, coach hiring, and major competition calls-to-action now point users toward the mobile experience.

↻

Mobile errors are recoverable

Important screens now distinguish “nothing here” from “could not load,” show useful messages, restore failed optimistic actions, and give the user a retry path.

⚙

The server is safer under real-world pressure

Concurrent game joins and roster changes are atomic, exact-slot booking collisions are guarded, malformed input receives clean errors, and venue deletion preserves history.
