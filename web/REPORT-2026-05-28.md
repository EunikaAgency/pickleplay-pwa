## API tasks (10 commits)

- **Cloned the API repo** (`jhonivancuaco/pickleballers-api`) into `/var/public/pickleplay/api/` and stood it up locally.
- **Set up MongoDB + imported real handoff data** — wrote `npm run db:import` (script at `src/shared/db/import-real-data.ts`) that reads the 5 CSVs in `real-data/handoff/`, drops target collections, and loads everything in: **179 venues**, **70 cities**, **815 venue hours**, **185 pricing blocks**, **32 coaches**. Re-runnable; bypasses Mongoose validators because the CSV is the source of truth.
- **Deployed via PM2 on port 9002** — `ecosystem.config.json` configured, autostart wired through the existing `pm2-eunika-blue.service` systemd unit. Public host `pickleballer-api.eunika.xyz` resolved via Cloudflare.
- **Added Playwright responsive-verify helper script** — captures the app at four viewport sizes for visual QA.
- **Restructured the API source to vertical slices** — moved `src/controllers/<Domain>/` + `src/models/X.model.ts` into `src/features/<feature>/{controller,routes,model}.ts`. Infrastructure (db, lib, middleware) collapsed into `src/shared/`. CoachReviews folded into the coaches feature; VenueManagement folded into venues. Fixed two pre-existing typecheck errors in the import script along the way.
- **Removed the `/sitemap.xml` endpoint** — it had wrong default URLs (referenced a non-existent host) and the wrong path templates. Premature SEO surface; can be revisited at launch from the actual public site.
- **Wired image serving end-to-end** — extended `db:import` to mirror `real-data/handoff/images/**` → `api/uploads/images/**` whenever the JPEG payload is present (idempotent, skips by size). Mounted `/images/*` via Hono's `serveStatic`. URLs already stored in Mongo as `/images/venues/<slug>/<file>` now resolve cleanly.
- **Built and ran the Google Drive image downloader** (`npm run db:download-images` at `src/shared/db/download-images.ts`) — extracts file IDs from `venues.csv` `main_image_url` + `gallery_image_urls` columns, follows Drive's export-download redirect with virus-scan-confirm fallback, saves with the canonical filenames from `venue_images.csv`. Bounded concurrency (4 parallel). First run downloaded **238 of 244 images successfully (97.5%)**.
- **Committed the 238 venue JPEGs as a canonical bundle** (~72 MB) so future installs don't depend on Drive URL availability.
- **Fixed CORS allowlist parsing** — the comma-separated `CORS_ORIGIN` env was being treated as a single literal origin, so no browser request ever matched. Now splits into an array and reflects each whitelisted origin individually; rejects unknown origins.
- **Relaxed Cross-Origin-Resource-Policy from same-origin to cross-origin** — `secureHeaders()` was blocking sibling-subdomain fetches with `ERR_BLOCKED_BY_RESPONSE.NotSameOrigin` even with correct CORS headers.
- **Added the admin account** directly into MongoDB via bcrypt-hashed insert: `info@eunika.agency` / `justinianthegreat!`, `roleDefault: 'admin'`.
- **Wrote the dummy user seed script** (`npm run db:seed:users`) — fetches profiles from randomuser.me and creates 32 coach users (each claims one of the imported Coach profiles via `User.coachId` ↔ `Coach.userId`), 12 venue owners (each assigned to a venue via `Venue.ownerUserId`), and 25 random players. Idempotent; cleans previous dummies on re-run.
- **Wrote `api/CLAUDE.md`** — documents the vertical-slice layout, the rule that `/lists` must stay in sync with mounted routes, the import-validation-bypass convention, and known TODOs.
- **Rebuilt `/lists` HTML endpoint** — was advertising paths like `/api/v1/coach-reviews`, `/media`, `/subscriptions`, `/admin` that returned 404 because they had no root handler. Now grouped by feature (12 sections: Root, Auth, Discovery, Venues, Coaches, Content, Bookings & Payments, Interactions, Media, Subscriptions, Venue management, Admin), with `auth` / `admin` lock chips. Clickable links only render for working public GETs; `/search` gets a sample `?q=zone` so it returns 200 on click. Plus `/` and `/health` converted to matching dark-theme HTML pages with nav links.
